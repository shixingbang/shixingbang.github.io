QQ信息加密（基础篇

本文是我为了混混实验钻研安全技术而对QQ的信息发送做的一次逆向，由于我技术有限，如果大家发现本文有哪里写的不好，还请大家来打我呀多多调指教～

引言

先介绍一下对软件进行逆向的一般流程吧，这样大家读这篇文章的时候也能大致明白我在做什么以及整体进行到第几步了。

  1. 拿到程序的可执行文件
  2. 对程序进行外部观察和处理
  3. 阅读头文件以及汇编代码
  4. 动态调试
  5. 编写动态库
  6. 注入

是不是看起来很简单呢 （说出来我自己都不信QAQ
好了不说废话了、开始对 iPhone 上的 QQ 进行逆向吧！ （各位系好安全带，我要发车了

提取可执行文件

这一步其实非常简单，ssh 到手机上，找到 QQ 所在目录然后 scp 到电脑上就好了，但是由于苹果应用商店对上架过的应用都加了一层壳，使之不能被直接反编译，所以我们要先砸下壳，操作大概是这样：（满脑子都是骚操作
>

1.获取 dumpdecrypted，这是一个用来砸壳的库，先下载

cd /Users/随便创个目录吧我就不乱起名了太尴尬了
git init
git clone git://github.com/stefanesser/dumpdecrypted/

2.编译它

cd yourpath/dumpdecrypted
make
这一步执行结束后，会在当前目录下生成一个dumpdecrypted.dylib文件，就是砸壳所需要的那个动态库了

3.找到 QQ 可执行文件所在位置, 先通过 ssh 连接到手机上：

ssh root@iOSIP
然后列举出所有进程
ps -e


上图中的 /var/mobile/Containers/Bundle/Application/CB131FB1-580F-4FA2-A6AE-561A3BB111D3/QQ.app/QQ 就是 QQ 可执行文件的全路径

然后我们还需要 QQ 的 Documents路径，需要另一个工具 cycript 来得到， 命令如下：
cycript -p QQ
[[NSFileManager defaultManager] URLsForDirectory:NSDocumentDirectory inDomains: NSUserDomainMask][0]

得到 Documents 目录路径后，将 dumpdecrypted.dylib 拷贝到目录下
scp /yourPath/dumpdecrypted/dumpdecrypted.dylib root@iOSIP:/DOCUMENTPATH

开始砸壳

cd /DOCUMENTPATH
DYLD_INSERT_LIBRARIES=dumpdecrypted.dylib /APPLICATIONPATH

生成 QQ.decrypted 后， 用 scp 将其拷回电脑

嗯步骤就是这些了，因为这部分太简单了 我太懒了 这些不是重点我就不每一步都配图了
注：/APPLICATIONPATH 是 ps -e 得到的那个路径、 /DOCUMENTPATH 是 cycript 里得到的那个路径

对程序进行观察和外部处理

  • 首先推荐一个叫 Reveal 的软件，配置好之后可以连接手机，打印出当前页面的各个控件层次结构，方便进行外部观察，效果图大概是这样：

    因为消息发送的函数一定会在聊天界面里被调用，所以我先在这个页面里观察了一下，大致了解了一下这个界面里控件的类名，诸如 QQChatViewTable, QQAIORichImageCellView.

  • 其次是 class-dump, 用它来获取一个可执行文件的所有类名， 命令如下
    class-dump -s -S -H QQ.dumpdecrypted -o /somePath
    得到 QQ 的所有头文件：

    以 QQChatViewController.h 为例，我们看到这个类实现了九个协议，包含了七十条属性以及二百四十个方法，不得不说 QQ 是个庞然大物，光一个类就有这么多东西，而整个 QQ 中有 12442 个类。真是吓坏宝宝了

  • 最后是 cycript， 用它来动态获取程序内部信息，比方说获取当前类类名。
    在注入进 QQ 后，我先用
    [[UIApp keyWindow] recursiveDescription].toString()
    打印出了当前界面的所有控件，然后在一坨东西里随便找了个 QQInputBar, 然后通过它的地址不断调用
    [#0x???????? nextResponder]

    最终得到当前界面 viewController 的类名： QQChatViewController

至此外部分析基本结束，已经判断出聊天界面的类名是 QQChatViewController, 由于消息在这个类里得到发送，所以这个类一定包含了消息发送的方法，下一步就是去头文件里找线索，然后用 lldb + debugserver 动态调试进行确认。

阅读头文件以及汇编代码

  • 首先是头文件，在 QQChatViewController 中重点看那些包含 send、 message 之类关键词的函数，我筛选出了这样几个函数：
    - (_Bool)doSendTextMsg:(id)arg1 showText:(id)arg2 isLocation:(_Bool)arg3;
    - (void)sendButtonClick:(id)arg1 msg:(id)arg2;
    - (_Bool)sendTextMsg:(id)arg1 showText:(id)arg2;
    这三个是和发送文本消息可能直接相关的函数，列入重点考察对象，其他一些发送位置，图片，data 之类的函数就先不管了。

动态调试

写得有点累，不说废话了，直接开始 lldb + debugserver 了。
先在手机 terminal 上开启 debugserver：
debugserver *:1234 -a "QQ"

电脑连接上去：
lldb
process connect connect:/iOSIP:1234

由于 ASLR （随机地址偏移） 的存在，每次程序运行时会有一个不同的基地址，我们用
image list -o -f
打印出程序本次运行的偏移， 最上面那个后缀 QQ 的就是，

可以看出本次 ASLR 值为 24000

然后为刚才找到的那几个函数打上断点，看看运行时会不会触发以判断它是不是要找的那一个。在 lldb 里为函数打断点需要知道函数的地址，地址计算公式是：
地址 = ASLR偏移 + 函数在程序中的地址
这里以 - (_Bool)sendTextMsg:(id)arg1 showText:(id)arg2;
为例为其打上断点：



输入 c 让程序继续运行，然后在消息发送的界面，输入一段文字后点击发送按钮，程序暂停住了，基本可以确定，这个函数里包含了消息发送。

附录

这里写一下本文用到的所有工具的配置方式：

  • theos
    export THEOS=/opt/theos
    sudo git clone git://github.com/DHowett/theos.git

  • cycript
    [官网] (http://www.cycript.org)

  • reveal
    不知道有没有的下了，可以去找找，我有空的话传一份到 github上？ （所以大概是没空了hhhh

  • class-dump
    [官网] (http://stevenygard.com/projects/class-dump)
    下载好后把 class-dump 复制到 /user/bin 下，然后在 terminal 执行
    sudo chmod 777 /usr/bin/class-dump

  • debugserver
    电脑端 scp root@iOSIP:/Developer/usr/bin/debugserver ~/debugserver
    lipo -thin arm64 ~/debugserver -output ~/debugserver
    下载 http://iosre.com/ent.xml 到 /Users/snakeninny 目录，然后
    /opt/theos/bin/ldid -Sent.xml debugserver
    将其拷回 iOS：
    scp ~/debugserver root@iOSIP:/usr/bin/debugserver
    ssh root@iOSIP
    在手机上为其赋予执行权限：
    chmod +x /user/bin/debugserver

  • hopper
    同 reveal 2333
